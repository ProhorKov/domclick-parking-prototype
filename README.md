# Домклик: машино-места

Кликабельный учебный React, TypeScript и Vite-прототип сервиса Домклик для покупки и продажи машино-мест.

Прототип показывает не просто каталог объявлений, а путь до безопасной сделки: поиск, карточка объекта, проверка Домклик, договор, расчёты, электронное оформление, финальный статус и кабинет партнёра для первичного рынка.

---

## 1. Что показывает прототип

### Основная идея

Покупатель машино-места хочет не только увидеть фото и цену. Ему важно понять:

- кто продаёт объект;
- можно ли доверять продавцу;
- есть ли ограничения;
- как будет подготовлен договор;
- как безопасно передать деньги;
- сколько стоит сопровождение сделки;
- что будет после отправки заявки.

Прототип отвечает на эти вопросы через продуктовый сценарий Домклик:

1. Пользователь ищет машино-место.
2. Открывает каталог с фильтрами.
3. Смотрит карточку объекта.
4. Видит блок проверки Домклик.
5. Считает стоимость сопровождения сделки.
6. Запускает оформление.
7. Проходит шаги сделки.
8. Получает финальный статус.

### Два рынка

| Рынок | Кто продаёт | Что важно показать |
| --- | --- | --- |
| Первичный | Застройщик, УК, партнёр или агент | Пул лотов, проверенный продавец, кабинет партнёра, аналитика спроса |
| Вторичный | Частный собственник | Проверка объекта и продавца, предупреждение о рисках, безопасные расчёты |

## 3. Запуск

### Установка

```bash
npm install
```

### Dev-сервер

```bash
npm run dev
```

Откройте URL из терминала. Обычно это:

```text
http://127.0.0.1:5173/
```

### Production-сборка

```bash
npm run build
```

Команда запускает TypeScript-проверку и собирает проект в `dist/`.

### Preview сборки

```bash
npm run preview
```

## 4. Если сайт не открывается

1. Проверьте, что терминал с `npm run dev` не закрыт.
2. Если сервер остановлен, запустите `npm run dev` заново.
3. Проверьте порт: Vite может открыть `5173`, `5174` или другой свободный порт.
4. Обновите вкладку браузера через `Ctrl + R`.
5. Если порт занят, остановите старый процесс через `Ctrl + C` и запустите сервер снова.

## 6. Главные файлы

### `src/App.tsx`

Управляет текущим экраном, выбранным лотом и рынком каталога. Роутера нет намеренно: для учебного кликабельного прототипа достаточно локального состояния.

Экраны:

- `home`;
- `catalog`;
- `detail`;
- `deal`;
- `partner`;
- `flow`;
- `smart`;
- `success`.

### `src/types.ts`

Содержит типы:

- `Market`;
- `Screen`;
- `VerificationState`;
- `DealServiceId`;
- `DealCalculatorMode`;
- `Lot`;
- `DealService`;
- `Navigate`.

### `src/data/lots.ts`

Содержит локальные лоты, статусы проверки, продавцов, цены, теги и данные для кабинета партнёра.

У каждого лота есть:

- `market`;
- `verification`;
- `image`;
- `imageAlt`.

### `src/data/mapAreas.ts`

Содержит зоны условной карты Москвы: ЖК, район, улицу и метро. У каждой зоны есть рынок, средняя цена, количество доступных машино-мест и связанные `lotIds`, по которым каталог ограничивает выдачу.

### `src/data/smartQuestions.ts`

Содержит вопросы и варианты ответов для сценария умного подбора: ключевые критерии, локацию, бюджет, сценарий сделки, рынок и причины обратной связи.

### `src/utils/recommendations.ts`

Содержит локальный скоринг `getSmartRecommendations(lots, answers, feedback)`. Функция не вызывает API и возвращает 3 рекомендации с причинами, совпавшими критериями, баллами и метками карточек.

### `src/data/dealServices.ts`

Содержит услуги калькулятора сделки:

| Услуга | Цена |
| --- | ---: |
| Регистрация сделки | 2 000 ₽ |
| Проверка объекта | 1 500 ₽ |
| Проверка продавца/покупателя | 3 000 ₽ |
| Безопасный расчёт | 1 500 ₽ |
| Продвижение объявления | 500 ₽ |

Также здесь лежат функции:

- `formatMoney(value)`;
- `calculateSeparateTotal(selectedServices)`;
- `calculateSaving()`.

### `src/components/DealCalculator.tsx`

Интерактивный калькулятор стоимости сопровождения сделки.

Он используется:

- в карточке объекта внутри правого sticky-блока;
- на экране сделки перед шагами;
- внутри шага `Расчёты`.

## 7. Калькулятор сделки

### Что умеет

Калькулятор показывает два варианта:

1. `Пакет безопасной сделки`.
2. `Собрать услуги отдельно`.

По умолчанию выбран пакет.

### Пакет безопасной сделки

Цена пакета:

```text
7 000 ₽
```

Пакет включает:

- регистрацию сделки;
- проверку объекта;
- проверку продавца/покупателя;
- безопасный расчёт;
- продвижение объявления.

Если покупать все услуги отдельно, сумма:

```text
8 500 ₽
```

Экономия пакета:

```text
1 500 ₽
```

В интерфейсе показывается подсказка:

```text
Пакет дешевле отдельных услуг на 1 500 ₽
```

### Отдельные услуги

Пользователь может переключиться на режим:

```text
Собрать услуги отдельно
```

В этом режиме можно включать и выключать чекбоксы услуг. Итоговая сумма пересчитывается автоматически.

Если пользователь выбрал все услуги отдельно, показывается подсказка:

```text
Пакет безопасной сделки дешевле на 1 500 ₽
```

### CTA калькулятора

В карточке объекта кнопка зависит от выбранного режима:

- `Оформить пакет за 7 000 ₽`;
- `Продолжить с услугами на X ₽`.

Нажатие ведёт в оформление сделки.

## 10. Полный сценарий демонстрации

### Быстрое демо за 5–7 минут

1. Откройте главную.
2. Покажите поисковый блок.
3. Выберите `От застройщика`.
4. Нажмите `Найти`.
5. В каталоге покажите фильтры слева и карточки справа.
6. Нажмите `Смотреть`.
7. В карточке объекта покажите:
   - статус проверки;
   - блок `Проверка Домклик`;
   - sticky-блок с ценой;
   - калькулятор сделки.
8. В калькуляторе переключитесь на `Собрать услуги отдельно`.
9. Снимите одну услугу и покажите пересчёт суммы.
10. Верните пакет и покажите экономию 1 500 ₽.
11. Нажмите `Начать оформление`.
12. Пройдите шаги:
   - Проверка;
   - Договор;
   - Расчёты;
   - Оформление.
13. На шаге `Расчёты` снова покажите калькулятор.
14. Нажмите `Отправить на оформление`.
15. Покажите финальный экран.

### Вторичный рынок

1. Вернитесь на главную.
2. Нажмите `От собственника`.
3. Откройте лот `ЖК «Северный парк»`.
4. Покажите статус `Нужна дополнительная проверка`.
5. Покажите, что Домклик не скрывает риск, а объясняет следующий шаг.

### Кабинет партнёра

1. В шапке нажмите `Партнёрам`.
2. Нажмите `Загрузить лоты`.
3. Нажмите `План паркинга`.
4. Нажмите `Заявки покупателей`.
5. Покажите метрики, таблицу и рекомендации.

### Схема сценария

1. В шапке нажмите `Сервисы`.
2. Покажите три роли:
   - покупатель;
   - Домклик;
   - продавец.
3. Нажмите `Пройти как покупатель`.
4. Вернитесь и нажмите `Открыть кабинет партнёра`.

## 11. Мобильная демонстрация

### Что проверить на телефоне

1. Главная не должна иметь горизонтальный скролл.
2. Поисковый блок должен быть вертикальным и удобным для пальца.
3. В каталоге фильтры открываются кнопкой `Фильтры`.
4. Карточки каталога становятся вертикальными.
5. Карта остаётся компактной, дополнительные зоны уходят в `Ещё районы`.
6. AI подбор проходится крупными кнопками в одну колонку.
7. Изображения сохраняют пропорции и не растягиваются.
8. В карточке объекта CTA виден быстро, но не перекрывает контент.
9. Калькулятор на мобильном компактный и раскрывается по нажатию.
10. Таблица в кабинете партнёра скроллится внутри контейнера.
11. Финальный экран помещается на телефоне без визуального развала.

### Проверяемые ширины

В этой итерации прототип подготовлен под ширины:

| Ширина | Что проверять |

| 360 px | Самая узкая мобильная демонстрация, отсутствие горизонтального скролла |
| 390 px | Типичный iPhone viewport |
| 414 px | Крупный мобильный viewport |
| 768 px | Планшет и narrow desktop |
| 1024 px | Планшет landscape / малый ноутбук |
| 1280 px | Desktop |
| 1440 px | Широкий desktop |

CSS-контрольные точки:

- `1040px`;
- `820px`;

## 13. Деплой на GitHub Pages

Проект подготовлен к публикации на GitHub Pages через GitHub Actions.

Почему выбран этот способ:

- проект собирается через Vite, поэтому GitHub должен сначала выполнить `npm ci` и `npm run build`;
- собранный сайт лежит в `dist/`;
- workflow `.github/workflows/deploy.yml` сам загружает `dist/` как Pages-артефакт;
- не нужно хранить собранные файлы в отдельной ветке `gh-pages`.

Vite настроен с:

```ts
base: "./"
```

Это важно для GitHub Pages: сайт будет открываться не из корня домена, а по адресу вида:

```text
https://<github-login>.github.io/<repository-name>/
```

Относительный `base` помогает корректно подгружать JS, CSS и локальные изображения на таком вложенном пути.

### Что уже добавлено

В проекте есть workflow:

```text
.github/workflows/deploy.yml
```

Он запускается:

- при push в ветку `main`;
- при push в ветку `master`;
- вручную через кнопку `Run workflow` в GitHub Actions.

Workflow делает четыре действия:

1. Забирает код из репозитория.
2. Ставит Node.js и зависимости через `npm ci`.
3. Собирает проект командой `npm run build`.
4. Публикует папку `dist/` в GitHub Pages.

### Как опубликовать первый раз

1. Создайте новый репозиторий на GitHub.

   Например:

   ```text
   domclick-parking-prototype
   ```

2. В локальном проекте выполните:

   ```bash
   git add .
   git commit -m "Prepare Domclick parking prototype for GitHub Pages"
   git branch -M main
   git remote add origin https://github.com/<github-login>/<repository-name>.git
   git push -u origin main
   ```

3. Откройте репозиторий на GitHub.

4. Перейдите:

   ```text
   Settings → Pages
   ```

5. В блоке `Build and deployment` выберите:

   ```text
   Source → GitHub Actions
   ```

6. Перейдите:

   ```text
   Actions → Deploy to GitHub Pages
   ```

7. Дождитесь зелёной галочки у workflow.

8. Готовый сайт будет доступен по адресу:

   ```text
   https://<github-login>.github.io/<repository-name>/
   ```

### Если сайт открылся без стилей

Проверьте три вещи:

1. В `vite.config.ts` должно быть:

   ```ts
   base: "./"
   ```

2. В GitHub Pages должен быть выбран источник:

   ```text

   ```

3. Последний workflow в `Actions` должен завершиться успешно.

## 14. Карта и умный подбор

### Эмулятор карты

На главном экране есть интерактивный блок `MapSearch`.

Он показывает условную карту Москвы без подключения Яндекс Карт, Google Maps или внешних API. Карта собрана из CSS-разметки: дороги, река, подписи районов, метро и кликабельные зоны.

Доступные зоны лежат в `src/data/mapAreas.ts`:

- ЖК `Зелёный квартал`;
- ЖК `Сокол`;
- район `Северный парк`;
- `Лётная улица`;
- метро `Аэропорт`.

У каждой зоны указаны:

- тип зоны;
- район;
- рынок `primary` или `secondary`;
- количество доступных машино-мест;
- средняя цена;
- связанные `lotIds`;
- координаты на условной карте.

При выборе зоны показывается мини-панель с названием, типом, количеством предложений и средней ценой. Кнопка `Показать предложения` открывает каталог с контекстом выбранной зоны.

### Сценарий демонстрации через карту

1. Открыть главную страницу.
2. Выбрать на карте `ЖК «Зелёный квартал»`, `Северный парк` или другую зону.
3. Нажать `Показать предложения`.
4. В каталоге показать заголовок вида `Машино-места: ЖК «Зелёный квартал»`.
5. Показать, что выдача ограничена связанными лотами.
6. Нажать `Сбросить выбор`, чтобы вернуться к обычному каталогу.

На mobile основные зоны остаются кликабельными, а дополнительные зоны доступны в блоке `Ещё районы`.

### Умный подбор

Сценарий `SmartCatalogScreen` доступен с главной страницы через кнопку `Подобрать с AI` и из каталога через кнопку `Подобрать варианты`.

Это не чат и не настоящий AI. Это управляемый опросник для защиты прототипа:

1. Пользователь выбирает до 3 ключевых критериев.
2. Отвечает на 4 уточняющих вопроса.
3. Получает 3 рекомендации.
4. Может открыть карточку объекта, перейти в обычный каталог или указать, что не понравилось.

Вопросы лежат в `src/data/smartQuestions.ts`.

### Логика рекомендаций

Скоринг находится в `src/utils/recommendations.ts`.

Функция:

```ts
getSmartRecommendations(lots, answers, feedback)
```

учитывает:

- выбранные критерии;
- бюджет;
- рынок;
- близость к дому или лифту;
- тип паркинга;
- электронную сделку;
- статус проверки Домклик;
- обратную связь после первой выдачи.

Функция возвращает карточки с метками:

- `Лучшее совпадение`;
- `Самый безопасный вариант`;
- `Выгодная цена`.

Для каждой карточки возвращаются причины рекомендации, совпавшие критерии и внутренний `score`.

### Как показать обратную связь “Не понравилось”

1. Пройти умный подбор до выдачи.
2. В блоке `Не понравились варианты?` выбрать причину, например `Слишком дорого`.
3. Появится сообщение `Учли: слишком дорого. Показываем варианты дешевле.`
4. Подборка перестроится: изменится порядок, причины рекомендации или набор карточек.
5. Если после жёсткого условия остаётся мало вариантов, экран предложит открыть весь каталог.

Также у каждой рекомендации есть кнопка `Не подходит`, которая перестраивает выдачу в режиме `Показать больше вариантов`.

### Сценарий демонстрации через AI подбор

1. На главной нажать `Подобрать с AI`.
2. Выбрать критерии, например `Юридическая безопасность`, `Рядом с лифтом`, `Можно оформить онлайн`.
3. Ответить на уточняющие вопросы.
4. Нажать `Подобрать варианты`.
5. Показать 3 карточки и объяснение `Почему показываем этот вариант`.
6. Открыть карточку объекта через `Смотреть`.
7. Вернуться и показать блок обратной связи.
8. Нажать `Открыть весь каталог`, чтобы показать альтернативный обычный путь.

## 15. Ограничения

Прототип учебный.

Он не:

- проводит настоящие сделки;
- отправляет реальные заявки;
- подключает платёжные API;
- проверяет реальные документы;
- использует реальные частные объявления;
- хранит персональные данные;
- подключает backend.

Логотип подключён как открытый брендовый ресурс. Иллюстрации парковок локальные и нейтральные.
